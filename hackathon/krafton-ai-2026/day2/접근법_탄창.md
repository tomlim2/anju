## 대기 중

### #4 ~~Test Prefix Exploitation~~ ❌ 불가 (64비트 전부 seed, exact eq 0개)
**핵심:** 테스트 시퀀스는 noise-free. W ≤ 64이므로 bits W..63은 XOR 규칙을 정확히 만족.
- W=10이면 54개 EXACT 방정식 (노이즈 0!)
- W=30이면 34개 exact 방정식
- W=64이면 0개 (worst case)
```
각 W=1..64에 대해:
  bits W..63이 어떤 offset set {d1..dS} ⊆ {1..W}로 정확히 생성 가능한지 체크.
  가능한 W와 offset set 찾기.
  해당 offsets로 training data 검증 (~80%).
```
파일: solve_prefix.py → results_prefix.json

### #5 MILP (Mixed Integer Linear Programming)
Gemini 추천. scipy 또는 PuLP 사용.
```
64개 이진 변수 a_d ∈ {0,1}, 2000개 방정식.
목적함수: 만족하는 방정식 수 최대화.
제약: sum(a_d) ≤ 16 (sparsity)
```
파일: solve_milp.py → results_milp.json

### #6 BM + Voting
```
2000개 시퀀스 각각에 Berlekamp-Massey 적용.
노이즈 때문에 대부분 실패하지만 운 좋은 시퀀스 존재 가능.
BM 결과(다항식)를 수집, 가장 빈번한 다항식 = 정답.
또는: 가장 짧은 LFSR 길이의 시퀀스들만 모아서 투표.
```
파일: solve_bm_vote.py → results_bm_vote.json

### #7 Goldreich-Levin Algorithm
LPN의 이론적 최적 알고리즘.
```
{-1,+1} 공간에서 heavy Fourier coefficient 탐색.
랜덤 부분집합 S ⊆ {1..64} 선택,
해당 부분집합의 XOR와 target의 상관 계산.
Binary search로 heavy coefficient 위치 좁혀감.
```
파일: solve_gl.py → results_gl.json

### #8 Overdetermined + Syndrome Decoding
```
100개 방정식 샘플 (64개보다 많이).
~20개가 오류 (20% noise).
Syndrome decoding: 64개로 풀고, 나머지 36개로 오류 탐지.
오류 행 제거 후 재풀이. 반복.
```
파일: solve_syndrome.py → results_syndrome.json

### #9 SAT/SMT Solver (Z3)
```
pip install z3-solver
64개 Bool 변수 + 2000개 XOR 제약 (soft).
최대 만족 가능한 할당 찾기 (MaxSAT).
또는: 랜덤 subset 800개 (80% clean) → hard 제약 → SAT.
```
파일: solve_sat.py → results_sat.json

### #10 Simulated Annealing
```
랜덤 offset set으로 시작.
이웃: offset 하나 추가/제거/교체.
에너지 = training error rate.
온도 스케줄에 따라 accept/reject.
빠르게 수렴해야 함 (수초).
```
파일: solve_sa.py → results_sa.json

### #11 Genetic Algorithm
```
인구: 100개 랜덤 offset set.
적합도: training accuracy (높을수록 좋음).
교차: 두 set의 offset 섞기.
돌연변이: offset 하나 랜덤 변경.
세대 반복. 80% 이상 나오면 정답.
```
파일: solve_ga.py → results_ga.json

### #12 Matrix Rank / Low-weight Codeword
```
2000×65 parity check matrix H 구성 (64 input cols + 1 output col).
Low-weight codeword 찾기 = 적은 수의 column 합이 0.
이것이 곧 tap polynomial.
```
파일: solve_rank.py → results_rank.json

### #13 Multiple Regression Targets
```
t=64 하나만 쓰지 말고 t=64..128 전부 사용.
단, t가 커질수록 input에 노이즈 전파.
t별로 독립 풀이 → 결과 투표.
```
파일: solve_multi_t.py → results_multi_t.json

### #14 Noise Filtering + Gaussian Elim
```
2000개 시퀀스에서 "가장 깨끗해 보이는" 시퀀스 선별.
기준: 자기 자신의 internal consistency (연속 비트 패턴).
상위 100개로 Gaussian elimination.
```
파일: solve_filter.py → results_filter.json
