# Day 2 Plan — SparseTap

**Problem:** SparseTap — Noisy XOR LFSR offset recovery + noise-free prediction (192 bits)

**핵심:** GF(2) 위 sparse LFSR recovery. 2000개 noisy 시퀀스에서 숨겨진 XOR offsets (≤16개, ≤64) 찾고, noise-free 테스트 시퀀스 192비트 예측.

---

## Agent-2 Analysis (Perspective A: GF(2) 대수적 접근)

### Problem Interpretation

Berlekamp-Massey / sparse BM으로 최소다항식 복원. 2000개 시퀀스에서 majority voting으로 노이즈 제거 후 exact recovery.

### Approach

- 각 시퀀스에 BM 적용 → 후보 다항식 수집
- 2000개 결과 majority voting → consensus polynomial
- 또는: 노이즈 제거된 "clean" 시퀀스 복원 후 단일 BM

### Risks + Alternatives

- 노이즈 20%로 BM 직접 적용 실패 가능 → 상관 분석 결과로 검증용 전환

---

## Agent-3 Analysis (Perspective B: 통계적 상관 분석)

### Problem Interpretation

각 후보 offset d에 대해 `seq[n] ⊕ seq[n-d]`의 bias를 2000개 시퀀스 평균으로 계산. 진짜 offset이면 bias ≈ 0.6 (noise 20%), 아니면 0.5. 단일 offset → 조합으로 확장.

### Approach

1. 단일 offset 스캔: d=1..64, 각 d에 대해 2000×(256-64) 샘플로 bias 계산
2. 유의미한 bias 가진 offset 후보 선별
3. 다중 offset 조합: greedy search로 XOR 조합의 bias 최대화
4. 최종 offset set 확정 → test prefix로 192비트 생성

### Risks + Alternatives

- 단일 offset bias만으로 S>2일 때 약할 수 있음 → pairwise/triplet 상관도 필요
- 조합 폭발 → greedy + pruning으로 제한

---

## Agent-4 Analysis (Perspective C: 수치적 최적화 — Gradient Descent)

### Problem Interpretation

offset 존재를 연속 weight로 relaxation → sigmoid parameterization → cross-entropy loss로 gradient descent. **제출 필수 요건** (보고서에 수치적 방법 1개 이상).

### Approach

1. 64차원 weight vector w ∈ R^64, sigmoid(w_d)로 offset 존재 확률 모델링
2. 예측: ŷ[n] = σ(Σ w_d · (2·seq[n-d]-1)) 식으로 연속 relaxation
3. Binary cross-entropy loss, Adam optimizer
4. 수렴 후 threshold → discrete offset set

### Risks + Alternatives

- GF(2) XOR를 연속으로 근사하는 게 까다로움 → 정확한 relaxation 설계 중요
- 상관 분석보다 느림 → 독립 검증 + 보고서 요건 충족이 주 목적

### 차별화 전략

#### 1. 암호학 연결 (핵심 차별점)
이 문제 = **Fast Correlation Attack** (Meier & Staffelbach, 1989) — stream cipher LFSR 복원 문제. 보고서에서 명시적으로 연결하면 "문제의 본질을 파악했다" 신호.

#### 2. Information-theoretic bound (서두에 배치)
2000 seq × 192 useful bits × (1-H(0.2)) ≈ 106K bits of information → 64개 후보 중 ≤16개 offset 복원에 충분한가? **왜 풀 수 있는지** 이론적으로 증명.

#### 3. {0,1} → {-1,+1} 변환 (수학적 구조)
XOR in {0,1} = 곱셈 in {-1,+1}: `(-1)^(a⊕b) = (-1)^a · (-1)^b`
- 상관 분석 → 단순 내적
- GD relaxation → 자연스러운 연속화
- Walsh-Hadamard transform 직접 적용 가능

#### 4. 실험 방법론
- Bootstrap confidence interval on offset detection
- 2000개 중 1000/1000 split → 독립 검증
- Offset별 p-value (단순 threshold가 아닌 통계적 유의성)

#### 5. 실패 분석의 질 (보고서 필수 항목)
단순 "안됐다"가 아닌 **왜 안 되는지 이론적 설명:**
- "BM이 noise 20%에서 실패: 최소다항식 차수 급격 증가"
- "단일 offset 상관만으로 S=16일 때 한계: marginal effect 약화"
